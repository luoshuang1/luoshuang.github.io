AI绘画大模型本地部署

本文参考教程从零开始，手把手教你本地部署Stable Diffusion AI绘画(Win最新版)_整合包还是自己部署-CSDN博客，部署Stable diffusion webui。

部署算法环境

conda -v

打出这条语句时会报错

(base) C:\Users\21211>conda -vusage: conda-script.py [-h] [-v] [–no-plugins] [-V] COMMAND …conda-script.py: error: the following arguments are required: COMMAND

原因是在运行 conda -v 命令时，缺少了必须跟随 -v 选项的命令部分。在使用 Conda 命令时，-v 选项通常用于增加输出的详细程度，但它必须跟随一个有效的命令，比如 install、update、create 等。

如果你想要查看 Conda 的版本，应该使用 --version 选项，而不是 -v：

conda --version

这条命令会显示当前安装在你系统上的 Conda 的版本号。如果你想要查看更详细的 Conda 安装信息，可以使用：

conda info

这条命令会提供关于 Conda 安装的详细信息，包括 Conda 的版本号、Python 的版本号以及其他相关的详细信息。

实际操作

提示词是一门学问，要写好提示词比较难，第一次上手操作发现输入提示词之后生成的图片一言难尽，注意提示词只能是英文。

见此，自然就起了到网上抄作业的念头，但是当我把其他大牛的提示词输入的时候再生成发现生成的图片质量还是一言难尽，细究原因，原来是没有使用模型，或者说没有使用与作者一样的模型，那么我们现在要想生成好看的图片自然就需要好的模型了。

模型下载推荐到c站。

还是一样的放到models文件夹里的Stable-diffusion文件夹里。

当再次想要创作时，只需以下几步

第一步，打开miniconda，

第二步，执行conda activate sdwebui，

第三步，执行D:

第四步，执行cd stable-diffusion-webui

第五步，执行webui-user.bat. 等待黑色小窗出现http://127.0.0.1:7860字样，

第六步，浏览器打开http://127.0.0.1:7860
